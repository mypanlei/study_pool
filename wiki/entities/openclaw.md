# OpenClaw

> 自托管的「个人操作系统式 AI 助手」— 从手机/聊天窗口发一条消息，就能在自己的机器或服务器上持续做事。

## 关键信息

- **类型**: 个人/团队 AI 助手平台
- **开发商**: openclaw.ai
- **核心定位**: personal assistant / messaging-native agent platform
- **默认入口**: 聊天应用（WhatsApp, Telegram, Discord, Slack, Signal, iMessage 等）
- **运行位置**: 自托管机器/服务器
- **是否 coding-first**: 否，偏 personal assistant

## 核心特色

| 特性 | 描述 |
|------|------|
| Chat-native | 以聊天应用为第一入口 |
| Always-on | 24/7 在线执行 |
| 浏览器控制 | 可浏览网页、填表 |
| 系统控制 | 文件、Shell、插件体系 |
| 持久记忆 | 跨渠道上下文记忆 |
| 技能体系 | 自写技能、插件扩展 |

## 与 Hermes Agent 的对比

| 维度 | OpenClaw | Hermes Agent |
|------|----------|--------------|
| 设计中心 | 个人助理体验 + always-on | Agent runtime + 学习闭环 |
| 入口模型 | 消息优先 | CLI 和消息平台同等 |
| 记忆风格 | 越来越懂你的助理 | 越来越会做事的 runtime |
| Provider | 多模型但非核心卖点 | 20+ provider 为核心能力 |
